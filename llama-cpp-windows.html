🛠️ llama.cpp en Windows

Importante: si quieres ahorrarte la compilación puedes instalar directamente llama.cpp usando Scoop.

Esta guía te ayudará a configurar tu entorno, compilar el código fuente de llama.cpp y configurar un atajo fácil para iniciar el servidor.

Requisitos Previos

  • Un sistema operativo Windows.
  • Una terminal o línea de comandos (CMD, PowerShell o Git Bash).
  • Scoop instalado (para la gestión de paquetes).

Paso 1: Instalar las Herramientas Necesarias

Asegúrate de tener instaladas las herramientas esenciales para el desarrollo y la compilación.

Configurar el Bucket de Scoop (si no lo has hecho)

Este comando añade el repositorio principal de Scoop.

scoop bucket add main

Instalar Git

Utilizamos Scoop para instalar la versión de Git.

scoop install main/git

Instalar CMake y MinGW

Instalamos las herramientas de compilación necesarias para el proyecto.

scoop install mingw
scoop install cmake

Paso 2: Clonar el Repositorio de llama.cpp

Descargaremos el código fuente del proyecto desde GitHub.

git clone https://github.com/ggerganov/llama.cpp

Paso 3: Compilar llama.cpp

Compilaremos el código fuente para obtener el ejecutable final.

Navegar a la carpeta del proyecto:

cd llama.cpp

Crear la carpeta de construcción (build):

mkdir build

Entrar en la carpeta de construcción:

cd build

Configurar la compilación con CMake: Usamos la configuración específica para MinGW.

cmake .. -G "MinGW Makefiles"

Compilar el proyecto: Ejecutamos el comando final para compilar el proyecto en modo Release.

cmake --build . --config Release

✅ Resultado de la Compilación

El ejecutable se encontrará en la siguiente ruta:

.\build\bin\Release\

Paso 4: Configurar el Inicio del Servidor (PowerShell)

Una vez que el ejecutable está compilado, configuraremos una función en PowerShell para que puedas iniciar el servidor fácilmente con un solo comando.

Abrir el perfil de PowerShell

Abre PowerShell y ejecuta el siguiente comando para abrir el archivo de perfil:

notepad $PSHOME\Profile.ps1

Pegar la función

Copia y pega la siguiente función en el Bloc de Notas que se abrió.

# Iniciar servidor
function llama-server { 
    C:\llama.cpp\llama-server.exe `
        -m C:\modelos\gemma-4-E2B-it-UD-Q4_K_XL.gguf `
        --host 0.0.0.0 `
        --port 8080 `
        --jinja `
        --no-mmap `
        --cache-type-k q8_0 --cache-type-v q8_0 `
        --spec-type ngram-mod `
        --spec-ngram-mod-n-match 24 --spec-ngram-mod-n-min 48 --spec-ngram-mod-n-max 64 `
        --temp 0.6 --top-p 0.95 --top-k 20 `
        --dry-multiplier 0.1 --dry-base 1.05 --dry-allowed-length 12 --dry-penalty-last-n 128 `
        -np 1
}

# Iniciar CLI 
function llama-cli { 
    C:\llama.cpp\llama-cli.exe `
        -m C:\modelos\gemma-4-E2B-it-UD-Q4_K_XL.gguf `
        --jinja `
        -cnv `
        --no-mmap `
        --reasoning off `
        --temp 0.6 --top-p 0.95 --top-k 20 `
        --dry-multiplier 0.1 --dry-base 1.05 `
        --color
}

Guardar, cerrar y recargar

Guarda el archivo del perfil, cierra el Bloc de Notas y reinicia tu sesión de PowerShell para que los cambios surtan efecto.

Paso 5: Iniciar el Servidor

Ahora que la función está configurada, cada vez que quieras iniciar el servidor, solo escribe en PowerShell:

llama-server

¡Listo! El servidor llama-server se iniciará automáticamente con los parámetros que definiste.
