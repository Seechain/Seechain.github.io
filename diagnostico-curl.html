Diagnóstico con curl

Guía práctica para diagnosticar servicios usando curl.

Qué es curl

curl es una herramienta de línea de comandos para transferir datos con URL. Soporta múltiples protocolos (HTTP, HTTPS, FTP, etc.) y es ideal para probar y diagnosticar servicios.

Opciones de Verbose

-v, --verbose - Muestra toda la conversación HTTP: headers, método, status, redirecciones. Es la opción más usada para diagnóstico.

-i, --include - Incluye los headers de respuesta en la salida.

-I, --head - Solo obtiene los headers, no el cuerpo de la respuesta.

-s, --silent - Modo silencioso, oculta la barra de progreso.

-o, --output - Guarda la respuesta en un archivo.

-w, --write-out - Imprime información específica al terminar (código de status, tiempo, etc).

Diagnóstico de Servicios

Verificar si un servicio está corriendo

curl -v http://localhost:8080

Muestra si el servicio responde y qué headers devuelve.

Ver headers de respuesta

curl -I http://localhost:8080

Muestra solo los headers sin descargar el cuerpo.

Ver método HTTP

curl -v -X POST http://localhost:8080/api/generate

Usa -X para especificar el método HTTP (GET, POST, PUT, DELETE).

Detectar redirecciones

curl -vL http://localhost:8080

-L sigue las redirecciones automáticamente.

Timeout y conexiones

curl --connect-timeout 5 http://localhost:8080
curl -m 10 http://localhost:8080

--connect-timeout: segundos para establecer conexión.
-m: segundos para completar la transferencia.

Ejemplos Prácticos

API de ejemplo

# Verificar si el servicio está corriendo
curl -v http://localhost:8080

# Ver estado de la API
curl -s http://localhost:8080/api/status

# Generar respuesta (ejemplo)
curl -s http://localhost:8080/api/generate -d '{"prompt":"Hola","stream":false}'

Servicios web

# Ver headers de un sitio web
curl -I https://ejemplo.com

# Ver respuesta completa con seguimiento de redirecciones
curl -vL https://ejemplo.com

# Ver tiempo de respuesta
curl -w "Tiempo: %{time_total}s\n" -o /dev/null -s http://localhost:8080

APIs REST

# GET request
curl -v http://localhost:3000/api/usuarios

# POST con JSON
curl -v -X POST http://localhost:3000/api/usuarios \
  -H "Content-Type: application/json" \
  -d '{"nombre":"Juan"}'

# Ver código de respuesta
curl -w "Status: %{http_code}\n" -o /dev/null -s http://localhost:8080

# Ver tamaño de respuesta
curl -w "Tamaño: %{size_download} bytes\n" -o /dev/null -s http://localhost:8080

Interpretar la Salida Verbose

*   Trying 127.0.0.1:8080...
* Connected to localhost (127.0.0.1) port 8080 (#0)
> GET / HTTP/1.1
> Host: localhost:8080
> User-Agent: curl/8.5.0
> Accept: */*
>
< HTTP/1.1 200 OK
< Content-Type: application/json
< Date: Tue, 21 Apr 2026 12:00:00 GMT
< Content-Length: 28
<
{...cuerpo de la respuesta...}
* Connection #0 to host localhost left intact
  • * Trying - Intentando conectar al host
  • * Connected - Conexión exitosa
  • > - Datos enviados al servidor (request)
  • < - Datos recibidos del servidor (response)
  • HTTP/1.1 200 OK - Código de estado

Códigos de Estado HTTP Comunes

  • 200 - OK, solicitud exitosa
  • 201 - Created, recurso creado
  • 301 - Moved Permanently, redirección permanente
  • 302 - Found, redirección temporal
  • 400 - Bad Request, solicitud inválida
  • 401 - Unauthorized, autenticación requerida
  • 403 - Forbidden, acceso denegado
  • 404 - Not Found, recurso no encontrado
  • 500 - Internal Server Error, error del servidor
  • 502 - Bad Gateway
  • 503 - Service Unavailable

Cheat Sheet

# Básico
curl http://localhost:8080
curl -v http://localhost:8080
curl -I http://localhost:8080
curl -s http://localhost:8080

# Métodos
curl -X POST http://localhost:8080
curl -X DELETE http://localhost:8080

# Headers
curl -H "Content-Type: application/json" http://localhost:8080
curl -H "Authorization: Bearer token" http://localhost:8080

# Datos
curl -d '{"key":"value"}' http://localhost:8080
curl --data-urlencode "mensaje=Hola mundo" http://localhost:8080

# Archivos
curl -o archivo.txt http://localhost:8080
curl -O http://localhost:8080

# Control
curl -m 10 http://localhost:8080
curl --connect-timeout 5 http://localhost:8080
curl -L http://localhost:8080

# Información
curl -w "Status: %{http_code}\n" -o /dev/null -s http://localhost:8080
curl -w "Tiempo: %{time_total}s\n" -o /dev/null -s http://localhost:8080
curl -w "Tamaño: %{size_download}\n" -o /dev/null -s http://localhost:8080
