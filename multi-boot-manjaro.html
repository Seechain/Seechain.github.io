Multi-boot Manjaro y Windows en Discos Separados

Esta guia esta disenada para multi-boot con cada SO en un disco separado.

Si tienes una laptop con solo un disco, o si deseas instalar un SO en su propia particion del mismo disco, esta guia no es para ti.

Esta guia asume que cada SO se instalara usando UEFI y no el modo CSM o Legacy de UEFI. No soporta el metodo tradicional de boot BIOS. Adicionalmente, todos los discos deben usar el esquema de particionado GPT.

Se asume conocimiento y algun entendimiento de estas terminologias. Sin eso, tendras lectura adicional antes de configurar un sistema multi-boot.

Desconectar Otros Discos

Al crear un escenario multi-boot, cada SO necesita estar protegido de los efectos de instalar otro SO. Se recomienda primero desconectar el disco que contiene un SO existente antes de instalar un segundo.

Es posible que un boot loader UEFI de un nuevo SO se instale en la ESP de otro disco. Desconectar otros discos durante la instalacion previene esto mientras asegura que cada SO permanezca lo mas autocontenido posible.

Con sus respectivos archivos de boot en la ESP de su propio disco, cada SO mantiene una cierta portabilidad, y puede ser removido en cualquier momento si se desea.

La Guia General

Primero desconecta cualquier disco de SO previo. Instala el nuevo SO (en modo UEFI) en un disco nuevo, y cuando este completamente funcional, reconnecta el disco del SO original. Configura el SO Linux primero en el orden de boot.

Paso a Paso

  1. Apagar, desconectar y desenchufar el computador.
  2. Desconectar cualquier disco de SO existente.
  3. Conectar la corriente nuevamente.
  4. Bootear con el instalador del SO (DVD/USB).
  5. Instalar el nuevo SO en el disco nuevo y configurar el SO.
  6. Apagar, desconectar y desenchufar el computador.
  7. Reconectar el disco del SO previo.
  8. Conectar la corriente nuevamente.
  9. Bootear al BIOS y seleccionar Manjaro como primera opcion de boot.
  10. Reiniciar y disfrutar.

Esto permitira que Grub se convierta en el boot loader principal para ambos Manjaro y Windows.

Advertencias Importantes

Secure Boot

Manjaro no soporta configuracion Secure Boot. Cada BIOS es diferente, asi que el procedimiento para deshabilitar Secure Boot puede variar. Consulta la documentacion de tu BIOS.

Si Secure Boot esta habilitado debido a un requerimiento de Microsoft (Windows 11), puede necesitarse una solucion alternativa para permitir que Windows bootee sin Secure Boot.

Secure Boot debe ser deshabilitado antes de instalar Manjaro.

Poner Manjaro Primero en el Orden de Boot

Bootear directamente a la configuracion del BIOS y asegurarse de que el disco con Linux/Grub/rEFInd (tu EFI principal) este seleccionado para bootear primero.

Windows Fast Startup

Por defecto, Windows 10/11 usan una variante de hibernacion que Microsoft llama Fast Startup. Windows en realidad no se apaga, sino que despierta de hibernacion dando la ilusion de un inicio mas rapido. Esto frecuentemente causa problemas con el boot y el montaje NTFS en Linux.

Fast Startup debe ser deshabilitado en un escenario multi-boot.

Para deshabilitar Fast Startup, ejecuta este comando desde un simbolo del sistema administrativo en Windows:

powercfg /h off

Fast Boot

Fast Boot es una opcion del BIOS que, si esta presente, permite saltarse ciertos autotest durante el pre-boot. Si Fast Boot esta habilitado, generalmente se recomienda deshabilitarlo en tu BIOS cuando usas Linux.

UTC vs Hora Local

Linux por defecto usa UTC en lugar de hora local; Windows usa hora local.

Esto causa problemas particulares al multi-boot, ya que cada SO por defecto usa un estandar de tiempo diferente. El metodo mas elegante es crear un archivo de registro de Windows llamado RealTimeIsUniversal.reg:

Windows Registry Editor Version 5.00

[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\TimeZoneInformation]
"RealTimeIsUniversal"=dword:00000001

Windows No Aparece en el Menu Grub

Windows puede no aparecer aun en el menu GRUB. El paquete os-prober es responsable de detectar otros sistemas operativos durante el boot.

Habilitar os-prober

Verificar que os-prober esta instalado:

pacman -Qi os-prober

Instalar si no esta:

sudo pacman -S os-prober

Verificar el estado de os-prober:

cat /etc/default/grub | grep -i 'OS_PROBER'

El resultado debe incluir esta linea:

GRUB_DISABLE_OS_PROBER=false

Si la linea esta comentada, descomentala:

sudo nano /etc/default/grub

Luego actualizar GRUB:

sudo update-grub

rEFInd UEFI Boot Manager

rEFInd es una solucion versatil y mas manejable para multi-boot con varios sabores de Linux.

rEFInd asume el rol del boot loader inicial, que a su vez chain-loadea otros boot loaders UEFI, incluyendo el GRUB efi-stub, el bootloader de Windows, MacOS y BSD.

Esto permite dejar cada GRUB intacto y manejado por su SO respectivo. Ademas, permite bypasear cualquier GRUB instalado y bootear el kernel-stub directamente.

Instalar rEFInd

Instalar el paquete:

sudo pacman -S refind

Usar la herramienta propia de rEFInd para instalarlo en la ESP:

sudo refind-install

La estructura de directorios sera similar a:

EFI
├─ BOOT
├─ Manjaro
└─ refind

Si rEFInd no aparece la primera vez, reiniciar al BIOS/UEFI y asegurarse de que rEFInd UEFI Boot Manager este primero en prioridad de boot.

Cambiar el Kernel que usa rEFInd

Manjaro Linux permite instalar multiples kernels.

sudo pacman -S manjaro-refind-installer

Este paquete permite seleccionar facilmente el kernel preferido que rEFInd usara.

Temas para rEFInd

rEFInd es un boot manager grafico y permite personalizacion con muchos temas disponibles.

Consulta la rEFInd Themes Collection para ver temas disponibles.
