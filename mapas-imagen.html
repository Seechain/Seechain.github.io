Mapas de Imagen - Diseño Web 2000s

El mapa de imagen (image map) fue una de las técnicas más utilizadas en los primeros años del diseño web. Apareció oficialmente con la especificación HTML 3.2, publicada en enero de 1997. Consistía en maquetar un sitio completo (fondo, botones, textos y logotipos) dentro de una única imagen PNG o GIF, creada con herramientas como Fireworks o Photoshop. Luego, en editores como Dreamweaver, se trazaban áreas invisibles sobre esa imagen para convertir cada elemento visual en un enlace funcional.

Aunque hoy está casi en desuso, los mapas de imagen representan un hito en la evolución del desarrollo frontend.

¿Cómo funcionaba exactamente?

  1. Una imagen contenedora (layout.png) alojaba todo el diseño: menú superior, barra lateral, galería, pies de página, incluso textos que no eran realmente texto sino píxeles pintados.
  2. Una etiqueta map definía el mapa, con un nombre único.
  3. Etiquetas area dentro del mapa: cada una definía una zona interactiva con atributos como:
    • shape="rect" (rectángulo), "circle" o "poly" (polígono para formas libres).
    • coords="x1,y1,x2,y2" (coordenadas exactas dentro de la imagen).
    • href="pagina.html" (el destino del enlace).
  4. La imagen se vinculaba al mapa mediante usemap="#nombre".
<img src="layout.png" usemap="#menu">
<map name="menu">
  <area shape="rect" coords="10,20,110,50" href="inicio.html" alt="Inicio">
  <area shape="rect" coords="120,20,220,50" href="productos.html" alt="Productos">
  <area shape="poly" coords="230,20,280,50,280,80,230,80" href="contacto.html">
</map>

Dreamweaver tenía una herramienta visual: dibujabas rectángulos, círculos o polígonos directamente sobre la imagen, y él generaba el código automáticamente.

¿Por qué se adoptó tan masivamente?

  • Control total del diseño: Lo que veías en Photoshop era exactamente lo que el usuario veía.
  • Tipografías imposibles: Podías usar cualquier fuente (no solo las seguras para web) porque todo era una imagen.
  • Efectos visuales complejos: Sombras, degradados, texturas y brillos.
  • Dreamweaver lo hacía fácil: No necesitabas saber coordenadas.

Desventajas

  • No responsive ni escalable: Si la imagen se redimensionaba o la pantalla era más pequeña, los enlaces se desplazaban.
  • Mantenimiento infernal:Un simple cambio textual requería Photoshop y edición manual del mapa.
  • SEO pésimo: Los buscadores veían una imagen con zonas, pero no tenían texto real asociado. El atributo alt en cada <area> ayudaba un poco, pero no era suficiente. Google no podía indexar frases dentro de los botones.
  • Accesibilidad nula: Los lectores de pantalla tenían grandes dificultades con múltiples áreas superpuestas o formas complejas.
  • Sin feedback visual Al pasar el ratón por encima, no había cambio de color, sombra o subrayado (a menos que usaras JavaScript).

Cuando el CSS mató a los mapas

Entre 2004 y 2008 llegaron varios golpes mortales:

  • CSS posicionamiento (position: absolute; float flex) permitió layouts reales con elementos HTML independientes.
  • Textos reales: se podían usar fuentes web, tamaños relativos (em, rem) y estilos CSS para botones.
  • Sprites CSS: varias imágenes en una sola hoja, pero con background-position, no con mapas de coordenadas frágiles.
  • Navegadores móviles: Safari en iPhone (2007) mostró que los mapas de imagen no funcionaban bien en pantallas táctiles pequeñas.

Casos de uso legítimos

Aunque abandonados para layouts generales, los image maps aún tienen dos o tres aplicaciones muy concretas:

  1. Infografías interactivas: Un mapa político donde cada provincia lleva a su ficha.
  2. Planos de museos o edificios: Donde cada sala o departamento es un <area> poligonal.
  3. Juegos retro o experimentos artísticos (Nostalgia o restricción técnica).

Incluso en esos casos, SVG es superior: permite escalado, estilos CSS, eventos JavaScript y mejor accesibilidad.

<!-- Alternativa moderna a image map -->
<svg viewBox="0 0 300 200">
  <polygon points="10,10 50,10 30,50" onclick="location.href='pagina.html'" />
</svg>
