Image Maps - Diseño Web Early 2000s

Si comenzaste a diseñar webs en la era de Flash, tablas anidadas y resoluciones de 800×600, seguramente recuerdes esta técnica: abrías Fireworks o Photoshop, maquetabas todo el sitio (fondo, botones, textos, logos) en una única imagen PNG o GIF, la exportabas, y luego, en Dreamweaver, trazabas rectángulos invisibles encima para convertir cada botón visual en un enlace real.

Eso era un mapa de imagen (image map), y fue una de las técnicas más populares entre 1998 y 2004. Hoy está casi olvidada, pero merece un lugar en la historia del diseño frontend.

¿Cómo funcionaba exactamente?

  1. Una imagen contenedora (layout.png) alojaba todo el diseño: menú superior, barra lateral, galería, pies de página, incluso textos que no eran realmente texto sino píxeles pintados.
  2. Una etiqueta <map> definía el mapa, con un nombre único.
  3. Etiquetas <area> dentro del mapa: cada una definía una zona interactiva con atributos como:
    • shape="rect" (rectángulo), "circle" o "poly" (polígono para formas libres).
    • coords="x1,y1,x2,y2" (coordenadas exactas dentro de la imagen).
    • href="pagina.html" (el destino del enlace).
  4. La imagen se vinculaba al mapa mediante usemap="#nombre".
<img src="layout.png" usemap="#menu" alt="Mi sitio web 2002">
<map name="menu">
  <area shape="rect" coords="10,20,110,50" href="inicio.html" alt="Inicio">
  <area shape="rect" coords="120,20,220,50" href="productos.html" alt="Productos">
  <area shape="poly" coords="230,20,280,50,280,80,230,80" href="contacto.html">
</map>

Dreamweaver tenía una herramienta visual: dibujabas rectángulos, círculos o polígonos directamente sobre la imagen, y él generaba el código automáticamente.

¿Por qué se adoptó tan masivamente?

  • Control total del diseño: Lo que veías en Photoshop era exactamente lo que el usuario veía. Sin sorpresas CSS entre navegadores.
  • Tipografías imposibles: Podías usar cualquier fuente (no solo las seguras para web) porque todo era una imagen.
  • Efectos visuales complejos: Sombras, degradados, texturas, brillos —todo funcionaba sin CSS3 ni SVG.
  • Dreamweaver lo hacía fácil: No necesitabas saber coordenadas; solo dibujar.

Desventajas

1. No responsive ni escalable

Las coordenadas son absolutas. Si la imagen se redimensionaba (o la pantalla era más pequeña), los enlaces se desplazaban. En móviles era un desastre.

2. Mantenimiento infernal

¿Cambiar un botón de "Servicios" a "Productos"? Había que:

  • Modificar el PNG original.
  • Reexportar.
  • Recalcular todas las coordenadas si la posición cambiaba mínimamente.
  • Actualizar el HTML.

Un simple cambio textual requería Photoshop y edición manual del mapa.

3. SEO pésimo

Los buscadores veían una imagen con zonas, pero no tenían texto real asociado. El atributo alt en cada <area> ayudaba un poco, pero no era suficiente. Google no podía indexar frases dentro de los botones.

4. Accesibilidad nula

Los lectores de pantalla tenían grandes dificultades con múltiples áreas superpuestas o formas complejas.

5. Sin feedback visual

Al pasar el ratón por encima, no había cambio de color, sombra o subrayado (a menos que usaras JavaScript).

Cuando el CSS mató a los mapas

Entre 2004 y 2008 llegaron varios golpes mortales:

  • CSS posicionamiento (position: absolute; float flex) permitió layouts reales con elementos HTML independientes.
  • Textos reales: se podían usar fuentes web, tamaños relativos (em, rem) y estilos CSS para botones.
  • Sprites CSS: varias imágenes en una sola hoja, pero con background-position, no con mapas de coordenadas frágiles.
  • Navegadores móviles: Safari en iPhone (2007) mostró que los mapas de imagen no funcionaban bien en pantallas táctiles pequeñas.

Casos de uso legítimos

Aunque abandonados para layouts generales, los image maps aún tienen dos o tres aplicaciones muy concretas:

  1. Infografías interactivas: Un mapa político donde cada provincia lleva a su ficha. Por ejemplo, en medios como El País o BBC, para resultados electorales.
  2. Planos de museos o edificios: Donde cada sala o departamento es un <area> poligonal.
  3. Juegos retro o experimentos artísticos (por pura nostalgia o restricción técnica).

Incluso en esos casos, SVG es superior: permite escalado, estilos CSS, eventos JavaScript y mejor accesibilidad.

<!-- Alternativa moderna a image map -->
<svg viewBox="0 0 300 200">
  <polygon points="10,10 50,10 30,50" onclick="location.href='pagina.html'" />
</svg>
