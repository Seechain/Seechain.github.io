Hosting detras de CGNAT con Cloudflare Tunnel

Tu ISP te puso detras de CGNAT y no puedes abrir puertos en tu router? No todo esta perdido! Puedes hostear sitios web y servicios publicos desde casa, gratis, sin abrir puertos, usando Cloudflare Tunnel.

Que es CGNAT?

CGNAT (Carrier-Grade NAT) es cuando tu proveedor de internet usa NAT compartiendote con otros clientes. No tienes tu propia IP publica - todos comparten la misma. Esto signifie que el trafico entrante no sabe llegar a tu servidor.

Tradicionalmente harias port forwarding en tu router, pero con CGNAT eso no funciona porque el router de tu ISP ni siquiera conoce tu servidor.

La solucion: Cloudflare Tunnel

Cloudflare Tunnel invierte la logica: en lugar de abrir puertos para que el mundo llegue a ti, tu servidor hace una conexion saliente a los servidores de Cloudflare. El traficopublico va a Cloudflare, que lo envia por el tunnel hasta tu casa.

Ventajas

  • No necesitas abrir puertos en el router
  • No necesitas IP publica estatica
  • Funciona detras de CGNAT
  • SSL automatico (Cloudflare maneja los certificados)
  • Proteccion DDoS integrada
  • Tu IP de casa queda oculta
  • Totalmente gratis

Requisitos

  • Una cuenta en Cloudflare (gratis)
  • Un dominio con DNS gestionado por Cloudflare
  • Un servidor Linux en tu red local con algum servicio web funcionando

Paso 1: Agrega tu dominio a Cloudflare

  1. Registrate en Cloudflare
  2. Agrega tu dominio (puedes comprar uno o usar uno que ya tengas)
  3. Cambia los nameservers en tu registrador a los que Cloudflare te indique

Paso 2: Crea el Tunnel

  1. Ve a Zero Trust Dashboard - Networks - Tunnels
  2. Click en Create Tunnel
  3. Selecciona Cloudflared como tipo
  4. Asigna un nombre al tunnel (ej: "mi-homelab")

Paso 3: Instala cloudflared en tu servidor

Cloudflare te dara un comando de instalacion. Ejemplo para Linux amd64:

wget -q https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb
dpkg -i cloudflared-linux-amd64.deb

O usa Docker:

docker run -d --name cloudflared \
  --network host \
  cloudflare/cloudflared:latest tunnel --no-autoupdate run <TOKEN>

El token lo obtienes del dashboard al crear el tunnel.

Paso 4: Configura el hostname publico

En el dashboard del tunnel, agrega un Public Hostname:

SubdomainTu eliges (ej: erp.midominio.com)
DomainTu dominio
ServiceHTTP en localhost:PUERTO

Ejemplo: si tienes Odoo en el puerto 8069 de tu red local:

  • Subdomain: erp
  • Service: http://192.168.1.100:8069

Ahora cuando alguien visite erp.midominio.com, Cloudflare enviara el trafico por el tunnel hasta tu servidor.

Paso 5: (Opcional) Agrega autenticacion

Puedes anadir Cloudflare Access para que nadie pueda ver tus servicios sin antes autenticarse (email, Google, GitHub, etc.).

Ve a Zero Trust - Access - Applications y agrega una politica de autenticacion.

Notas finales

  • Tu servidor debe permitir salida por TCP/UDP puerto 443
  • El tunnel es persistente, pero conviene configurarlo en un contenedor Docker o como servicio para que arranque automaticamente al iniciar
  • Puedes exponer multiples servicios con un solo tunnel usando reglas de ingress
