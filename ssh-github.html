SSH con GitHub

Guía paso a paso para configurar SSH y conectarte a GitHub.

Qué es SSH

SSH (Secure Shell) es un protocolo que permite conexiones seguras entre tu computadora y servidores remotos. Con SSH puedes autenticarte con GitHub usando una clave criptográfica en lugar de usuario y contraseña.

Por qué usar SSH

  • No necesitas escribir token ni contraseña cada vez que haces push
  • Más seguro que autenticación por contraseña
  • Una vez configurado, funciona automáticamente
  • Puedes automatizar tareas con git

Algoritmo Ed25519

Ed25519 es el algoritmo recomendado por GitHub para claves SSH:

  • Más seguro que RSA
  • Más rápido
  • Claves más cortas
  • Soportado oficialmente por GitHub

Paso 1: Generar la clave SSH

Ejecuta este comando en tu terminal:

ssh-keygen -t ed25519 -C "tu@email.com"

Qué significa:

  • -t ed25519 - Usa el algoritmo Ed25519
  • -C "tu@email.com" - Un comentario para identificar la clave

El comando te preguntará:

1. Where to save the key: Presiona Enter para aceptar la ubicación por defecto (~/.ssh/id_ed25519)

2. Passphrase: Deja vacío presionando Enter (o escribe una contraseña si quieres protección extra)

3. Confirm passphrase: Presiona Enter de nuevo

Deberías ver algo como:

Generating public/private ed25519 key pair.
Your identification has been saved in /home/user/.ssh/id_ed25519
Your public key has been saved in /home/user/.ssh/id_ed25519.pub

Paso 2: Añadir la clave a GitHub

Primero, copia tu clave pública:

cat ~/.ssh/id_ed25519.pub

Debería mostrar algo que empieza con ssh-ed25519 y termina con tu email.

Ahora en GitHub:

1. Ve a Settings (tu foto de perfil > Settings)

2. En el menú lateral busca "SSH and GPG keys"

3. Click en "New SSH key"

4. En "Title" escribe algo como "Mi computadora"

5. En "Key" pega todo el contenido de tu clave pública

6. Click en "Add SSH key"

Paso 3: Verificar la conexión

Prueba que todo funciona:

ssh -T git@github.com

Si es la primera vez, te preguntará si quieres continuar. Escribe "yes" y presiona Enter.

Deberías ver:

Hi tu-usuario! You've successfully authenticated.

Paso 4: Cambiar remote de HTTPS a SSH

Si ya tienes un repositorio clonado con HTTPS, cámbialo a SSH:

git remote set-url origin git@github.com:usuario/repo.git

Para saber tu URL SSH correcta:

  • En GitHub, ve a tu repositorio
  • Click en el botón verde "Code"
  • Selecciona "SSH"
  • Copia la URL que empieza con git@github.com:

Verifica el cambio:

git remote -v

Debería mostrar:

origin  git@github.com:usuario/repo.git (fetch)
origin  git@github.com:usuario/repo.git (push)

Paso 5: Automatizar SSH Agent (opcional)

Para no tener que ejecutar ssh-add cada vez, añade esto al final de tu archivo ~/.bashrc:

SSH_ENV="$HOME/.ssh/agent.env"

function start_agent {
    # El comando 'ssh-agent' sin ruta absoluta funciona en cualquier distro
    ssh-agent -s | sed 's/^echo/#echo/' > "${SSH_ENV}"
    chmod 600 "${SSH_ENV}"
    . "${SSH_ENV}" > /dev/null
    ssh-add ~/.ssh/id_ed25519
}

if [ -f "${SSH_ENV}" ]; then
    . "${SSH_ENV}" > /dev/null
    ps -ef | grep ${SSH_AGENT_PID} | grep [s]sh-agent > /dev/null || {
        start_agent
    }
else
    start_agent
fi

Después reinicia tu terminal o ejecuta:

source ~/.bashrc

Solución de problemas

Error: Permission denied (publickey)

  • Verifica que añadiste la clave correcta en GitHub
  • Ejecuta ssh -vvv git@github.com para ver detalles del error

Error: Could not resolve hostname

  • Verifica tu conexión a internet
  • Verifica que escribiste bien la URL

La clave no se añade automáticamente

  • Ejecuta manualmente: ssh-add ~/.ssh/id_ed25519

Comandos útiles

# Ver claves disponibles
ls -la ~/.ssh

# Ver clave pública
cat ~/.ssh/id_ed25519.pub

# Probar conexión
ssh -T git@github.com

# Ver todas las claves añadidas al agente
ssh-add -l
